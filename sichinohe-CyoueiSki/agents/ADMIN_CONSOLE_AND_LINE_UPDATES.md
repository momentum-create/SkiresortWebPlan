# 運営向け更新コンソール／LINE 自動更新（要件）

販売後に**貴社運営で更新する**ケースと、現場（ゲレンデ側）が**素人でも迷わない**ことを最優先にした設計メモ。公開サイト本体とは分離した **更新専用 URL**（例: `admin.example-resort.jp` または `/studio` + 認証）を前提とする。

---

## 設計原則（難易度を下げる）

1. **1画面＝今日やること** — 「朝イチ更新」「終業時クローズ」の2モードより開始。長文CMSではなくトグル・選択肢・数値のみ。
2. **説明文は現場語彙** — 「operations_summary」「enum」は内部IDのみ。UIは「動いてます／止まってます／確認中」など。
3. **プリセット** — 「強風で上部のみクローズ」「全日運転」「ナイターのみ」のワンタップ（内容は施設ごとに設定）。
4. **スマホファースト** — 手袋前提ではなく、屋外でも読める大きなタップ領域。
5. **Undo／直近履歴** — 誤タップで公開被害が出ないよう、直前1件の取り消しまたは「公開予約（確認ダイアルログ）」を標準装備。
6. **権限** — `viewer`（見るだけ）／`editor`（更新）／`owner`（LINE連携・ユーザー招待）。小規模施設は最初 editor のみでも可。
7. **欠損は許容** — 未入力項目はサイト上では「―」「確認中」とし、推測で自動入力しない。

## 更新項目グループ（`DATA_MODEL_NORMALIZED_RESORT.md` との対応）

| グループ | 運営が触るUIラベル例 | 入力タイプ |
|----------|----------------------|------------|
| 今日の営業 | 開園・終了予定、延長・早締め | 時刻プリセット＋トグル |
| 雪 | 積雪(cm)、24h降雪(cm)、コメント一言 | 数値＋短文（任意） |
| リフト | 各リフト 運転中／停止／確認中、停止理由 | リスト＋理由チップ |
| コース | （取得できる場合）開放／閉鎖 | 同上 |
| レストラン・売店 | 稼働／限定／終了 | アウトレット単位トグル＋時刻 |
| 託児 | 開／閉／要予約 | 単一選択 |
| スクール | 当日受付 受付中／締切／情報なし | 単一選択 |

詳細フィールドは「詳細を開く」アコーディオンに退避し、デフォルトは上記だけを見せる。

## LINE 連携の狙い（将来フェーズ）

現場が更新ページを開かず、**LINE に送るだけで同じデータストアが更新される**ようにする。公開サイト・japowserch へは既存パイプライン（Webhook／バッチ）と同一の正規JSONへ書き込む。

### 推奨アーキテクチャ（論理）

```
LINE Official Account（Messaging API）
        │ HTTPS Webhook（署名検証必須）
        ▼
「LINE ブリッジ」サービス … 送信者検証・レート制限・監査ログ
        │
        ├── 構造化コマンド解析（例: 「雪30」「リフト第一ペア 停」）
        └── または NLU（誤解釈リスクあり）→ 必ず確認リプライ
        ▼
正規化ストア（CMS／API）… 管理画面と同一ソース
        ▼
公開サイト・japowserch
```

### 運用パターン（安全順）

| 方式 | 長所 | 短所 |
|------|------|------|
| A. リッチメニュー＋Postback | 誤送信が少ない | 実装やや重い |
| B. 定型短文コマンド（厳密パース） | 実装軽い | 覚える必要 |
| C. LINE で URL を送り **LIFF** でミニフォーム | UX最高・誤変換なし | LIFF メンテ |

**推奨**: MVP は **C（LIFF）または A（リッチメニュー）** を優先し、自由文のみの自動反映は避ける（誤情報公開のリスクが高い）。

### セキュリティ最低ライン

- Webhook は **LINE 署名（`X-Line-Signature`）検証**必須。
- 送信可能な **LINE User ID をホワイトリスト**（または公式アカウントの権限付与済みユーザーのみ）。
- **監査ログ**（誰が・いつ・何を変更したか）を管理画面と同一ストアに。
- レート制限・異常パターンアラート（同一文の連投など）。

---

## 販売パッケージとの関係

- **自社運用**: 貴社がコンソール権限を保持し、テナント（ゲレンデ）は editor を招待するモデル。
- **サブスク**: LINE Official アカウントの料金・メッセージ通数、ブリッジのホスティングは原価として試算に含める（エージェント10と整合）。

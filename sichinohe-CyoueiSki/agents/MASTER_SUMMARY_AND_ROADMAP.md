# 七戸町営スキー場プロジェクト — 総まとめ（項目一覧）とロードマップ

実装フェーズの優先順（LINE連携）は **Phase1 Webコンソール → Phase2 LIFF → Phase3 リッチメニュー／定型送信** で固定。

**リフトマップ（見える化）を最優先する場合**: リポジトリ直下 [`docs/ROADMAP_LIFT_MAP_VISUAL_FIRST.md`](../../docs/ROADMAP_LIFT_MAP_VISUAL_FIRST.md) を主ロードマップとし、本書の H-1 以降と並行する。

## A. 戦略・ポジショニング（戦略レポート由来）

| # | 項目 |
|---|------|
| A1 | **トランジット・リゾート** — 東北新幹線「七戸十和田駅」から二次交通が短く、広域周遊（本州〜北海道縦断文脈）での「途中のパウダー」を訴求 |
| A2 | **時間対効果・摩擦低減** — 移動ストレスを減らし、ショートステイでも納得できる体験設計（リフト券の心理的ハードル等のナラティブ） |
| A3 | **二極ターゲット** — 上部非圧雪・急斜面（コア／パウダー）と下部ポニーリフト周辺の緩斜面・圧雪（初心者・ファミリー）を**空間的・時間的に棲み分け**して訴求 |
| A4 | **競争の少ないパウダー** — 小規模・知名度を「パウダーパニック回避」の文脈で語る（条件付き・誠実に） |
| A5 | **ストームデイ・バックアップ** — 周辺エリアの悪天候時に当ゲレンデを選択肢として示す（運行・視界は事実ベース） |
| A6 | **ローカルオーセンティシティ** — 町営ヒュッテ、木曜ナイター無料講習、周辺温泉など **公式確認・免責つき** でジャーニーとして紹介 |
| A7 | **局地デジタル** — マイクロウェザー連動・長尾SEO（高意図クエリ向けLP）、無加工POVの運用フック |
| A8 | **所在地** — 青森県七戸町（北海道ではない） |

## H. 七戸サイト「完成」までのロードマップ（フェーズ別）

「完成」の定義を **プレビュー公開できる公式MVP（情報の箱＋今日の手動更新の運用ルール）** と **フルプロダクト（ハブ連動・コンソール本番・LINE・アフィリエイト）** に分ける。

### H-0. あなたの指定「1日」以内に現実的に終えるスコープ（強く推奨）

**ゴール**: ステージングまたは本番URLで、来場判断に効く **トップ＋今日＋アクセス＋料金骨子** が見られ、スマホで読める。

| 時間目安 | フェーズ名 | やること（チェックリスト） |
|----------|------------|---------------------------|
| 0.5h | **D0 準備** | リポジトリ作成・ホスティング決定（Vercel／Cloudflare Pages 等）・ドメインDNSは後回し可 |
| 1h | **D1 骨格** | ルーティング・レイアウト・共通ヘッダ／フッタ・モバイルナビ |
| 2h | **D2 トップ** | ヒーロー・二極CTA・「今日」サマリー（**静的またはCMSの単一JSON／MD**・更新手順メモ） |
| 1.5h | **D3 コアページ** | `/access`（駅〜タクシー・注意書き）・`/tickets-rental`（表はプレースホルダ可、「要確認」明示）・`/courses`（画像1枚＋暫定テーブル） |
| 1h | **D4 信頼・問い合わせ** | `/contact`・安全の短文・電話／フォームのどちらか一方 |
| 1h | **D5 SEO最小** | title/description・OGP・`SkiResort` 相当の JSON-LD はテンプレ1本 |
| 1h | **D6 デプロイ** | CI・本番ビルド・404・軽い Lighthouse 確認 |

**1日で切る（翌日以降へ）**: 長尾LP3本の全文、英語全ページ、管理コンソール実装、LINE、japowserch API 連携、アフィリエイト埋め込み、リフト個別リアルタイムのDB駆動。
